2. Constraint DSL (Domain Specific Language)

Chuẩn hóa tuyệt đối ngôn ngữ cấu hình Rule để PostgreSQL, Neo4j và Python Engine hiểu chung một ngữ nghĩa.

{
  "rule_id": "HTN_SODIUM_LIMIT_V2",
  "rule_type": "LIMIT",      // Enum: LIMIT, EXCLUDE, REQUIRE, RECOMMEND
  "target_type": "NUTRIENT", // Enum: FOOD, INGREDIENT, CATEGORY, NUTRIENT
  "target_key": "N_SODIUM",
  "operator": "<=",          // Enum: <=, >=, ==, !=
  "value_expression": {
    "type": "STATIC",        // Enum: STATIC, FORMULA
    "value": 1500,
    "unit": "mg"
  }
}

3. Category Hierarchy & Ontology Rules

Định nghĩa cấu trúc phả hệ phân loại (DAG - Directed Acyclic Graph).

  • Multiple Parents: Cho phép (VD: Tomato thuộc Fruit VÀ Vegetable).
  • Max Depth: 5 levels (để tối ưu performance khi query Cypher -[:BELONGS_TO*1..5]->).
  • Cycle Prevention: Bắt buộc validate thuật toán Topo-sort tại tầng Application trước khi Insert vào Neo4j.

4. Canonicalization & Merge Strategy

Cơ chế chống rác và trùng lặp dữ liệu (De-duplication) từ nhiều nguồn.

erDiagram CANONICAL_FOOD ||--o{ ALIAS : "has_alias" CANONICAL_FOOD ||--o{ ENRICHMENT_SOURCE : "merged_from" CANONICAL_FOOD { uuid food_id PK string canonical_name "e.g., Pho Bo" } ALIAS { string name "Phở Bò, Beef Pho, Noodle Soup" }

Merge Strategy (Conflict Resolution): Áp dụng Weighted Confidence. Nếu có xung đột về Serving Size: USDA (1.0) > Manual (0.9) > LLM (0.7) > CV (0.5).

5. Projection Sync Architecture (Event-Driven)

Đồng bộ dữ liệu từ Source of Truth (Postgres) sang Read Model (Neo4j).

sequenceDiagram participant Admin participant API as KG API (Postgres) participant EventBus as Message Broker participant Worker as Projection Worker (Neo4j) Admin->>API: Approve Draft Food API->>API: Update Postgres state = VERIFIED API->>EventBus: Publish [FoodVerifiedEvent] EventBus->>Worker: Consume Event Worker->>Worker: Build Cypher Query Worker->>Worker: Cypher Merge (Idempotent) Worker-->>EventBus: Ack Message

8. Human Review Workflow (State Machine)

Xử lý quyết định của Admin/Medical Expert với các món ăn do LLM/CV sinh ra (Score 0.5 - 0.8).

  • APPROVE: Đổi state thành VERIFIED, push event sync Neo4j.
  • REJECT: Đổi state thành REJECTED (giữ lại trong DB để làm negative label train ML sau này).
  • APPROVE_WITH_EDIT: Tạo version mới, override dữ liệu của LLM, set source = MANUAL (Confidence = 0.9), push event sync.

9. Testing Strategy (Pyramid)

Do tính phức tạp của Knowledge Graph, chiến lược test được chia làm 4 lớp:

  • Lớp 1 - Unit Tests (Core Domain): Test DSL Parser, Formula Engine, Unit Converter (đảm bảo 1g = 1000mg). Không mock DB.
  • Lớp 2 - Integration Tests (Persistence): Test Projection Sync Worker đảm bảo Idempotent (chạy Cypher 2 lần không sinh node trùng).
  • Lớp 3 - Ontology Tests (Graph): Viết test phát hiện Cycle (Vòng lặp) trong Category Hierarchy.
  • Lớp 4 - E2E Tests (Constraint): Giả lập Use Case: "User có Seafood Allergy + Món ăn chứa Shrimp → API phải trả CRITICAL".
